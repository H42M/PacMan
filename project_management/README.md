# Project management evidence

This directory contains the project-management evidence for the 42 PacMan activity.

The team used a lightweight feature-branch workflow supported by task tracking, a branch plan, decision notes, risk analysis, acceptance tests, and a bug log. The goal was to keep the process useful for development and defense without turning the project into unnecessary bureaucracy.

## Files

- [`timeline.md`](timeline.md): project phases, planned vs actual progress, and a visual timeline.
- [`progress_tracking.md`](progress_tracking.md): milestone/Kanban-style progress summary.
- [`decisions.md`](decisions.md): key technical and organization decisions.
- [`risks.md`](risks.md): main project risks and mitigations.
- [`team_organization.md`](team_organization.md): team responsibilities, branch ownership, and review rules.
- [`test_plan.md`](test_plan.md): acceptance-test checklist for defense-critical features.
- [`bug_log.md`](bug_log.md): notable bugs, fixes, and remaining known limitations.
- [`final_report.md`](final_report.md): final project status and defense summary.
