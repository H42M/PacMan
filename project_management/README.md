# Project management evidence

This directory contains the project-management evidence for the 42 PacMan activity.

The team used a lightweight feature-branch workflow supported by task tracking, branch planning, decision notes, risk analysis, acceptance tests, bug tracking, and final reporting. The goal was to keep the process useful for development and defense without turning the project into unnecessary bureaucracy.

## Files

- [`timeline.md`](timeline.md): project phases, planned vs actual progress, and visual timeline.
- [`progress_tracking.md`](progress_tracking.md): milestone/Kanban-style progress summary and branch status.
- [`decisions.md`](decisions.md): key technical and organization decisions.
- [`risks.md`](risks.md): main project risks, mitigations, and final status.
- [`team_organization.md`](team_organization.md): team responsibilities, branch ownership, and review process.
- [`test_plan.md`](test_plan.md): acceptance-test checklist and final validation route.
- [`bug_log.md`](bug_log.md): notable bugs, fixes, and accepted limitations.
- [`final_report.md`](final_report.md): final project status and defense summary.

## Final status

The project reached a complete playable and packaged state. Core gameplay, UI/gameflow, highscores, level progression, cheat mode, project evidence, README documentation, and PyInstaller Linux packaging were completed. The final Linux build was uploaded to Itch.io for evaluation/demo purposes:
